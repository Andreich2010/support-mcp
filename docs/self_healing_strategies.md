# Стратегии Self-Healing для автоматического решения тикетов

Этот документ описывает типовые действия, которые может предпринять система для самостоятельного устранения ошибок.

---

## 1. Ошибки подключения к базе данных
- проверить корректность переменных окружения;
- предложить пользователю сверить пароль и логин;
- проверить доступность порта 5432;
- предложить команду для ручной проверки подключения.

---

## 2. Ошибки миграций
- предложить выполнить migrate;
- проверить наличие новых моделей;
- указать на необходимость обновления контейнера.

---

## 3. Ошибки конфигурации Docker
- проверить соответствие настроек .env и docker-compose.yml;
- предложить пересоздать контейнеры.

---

## 4. Ошибки кода
- указать строку ошибки;
- предложить возможную причину;
- предложить шаги отладки.

---

Документ используется для принятия решений в инструменте auto_resolve_ticket.
